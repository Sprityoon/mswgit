# MSW Conductor — 지휘자 운영 절차

상세 규약은 [docs/agents/conductor-role.md](../../../docs/agents/conductor-role.md)가 원본이다. 이 스킬은 그 실행 체크리스트.

## 1. 현황 재구성 (세션 시작마다 — 채팅 기록은 이어지지 않는다)

1. `docs/agents/subagent-handoff.md` §3 상단 실행 계획 블록 + 각 티켓 상태를 Read.
2. `game_design.md` Phase 트래커 §를 Grep으로 찾아 Read (84KB+ — 전체 읽기 금지).
3. `git log --oneline -10`으로 최근 커밋 보조 확인. 무보고 커밋(큐에 없는 작업)이 있으면 실사 후 소급 티켓 발행.

## 2. 보고 검수 (완료 보고가 들어온 티켓)

- **보고 3종 확인**: ① 채팅 요약 ② §3 상태 갱신 ③ `docs/agents/reports/T<n>-*.md` 파일 존재+검증 근거(refresh Error 수, 로그 발췌). **하나라도 없으면 [반려]** — 상태를 되돌리고 재작업 티켓화.
- Acceptance 전부 충족 전에는 `[완료]` 금지. 부분 완료는 검증 수준 병기: `[코드 완료 | 런타임 검증 보류(제작자)]`.
- Play 검증은 제작자(사람) 몫 — 로그·보고 근거 없이 "동작 확인" 기록 금지.
- 검수 통과 티켓: §2 완료 기록에 포인터 요약 이동 (후속 작업이 참조하는 스펙 원문은 §3 유지 가능).

## 3. 의사결정 기록

- 보류 항목("보스와 합의") 확정, 하드코딩 예외 승인/기각, 우선순위 배열.
- 모든 결정에 날짜 필수: `⚖️/🧭 YYYY-MM-DD 보스 확정` 형식.
- 판단 근거가 부족하면 제작자에게 질문 — 임의 확정 금지.

## 4. T티켓 발행

- **T번호는 단조 증가·재사용 금지** (큐 최대 번호 + 1).
- 필수 3요소: **Target(파일) / Change(변경) / Acceptance(완료 기준)** + 충돌 주의(레인 소유권).
- 하위 에이전트가 임의 판단할 여지(모호한 스펙, 미확정 의존성)를 제거한 뒤 발행.
- 병렬 배치면 레인별 소유 파일을 명시하고 겹침을 제거. `PersistenceManager`/`PlayerInventory`/`PlayerController` 등 공유 파일은 배치 내 순차 강제.
- `game_design.md`(해당 시스템 § + Phase 트래커)에 결정·상태 동기 반영.

## 5. 킥오프

표준 프롬프트 (T번호만 교체해 전달):

```
너는 구현자 역할이야. msw-worker 스킬(없으면 docs/agents/subagent-handoff.md §1 전체)을 로드하고
§3의 T<n>만 수행해. 소유 파일 밖 수정 금지, 완료 시 보고 3종(채팅 요약 + §3 상태 갱신 +
docs/agents/reports/T<n>-*.md) 필수. 검증은 티켓 완료 즉시 refresh 1회 이상 — Error 수를 보고서에 기재.
Play 검증은 범위 밖(제작자 수행) — "런타임 검증 보류"로 보고해.
```

## 금지 (지휘자가 하지 않는 일)

- 코드(`.mlua`)/데이터셋/모델/UI 직접 수정 — 편집 대상은 기획·운영 문서(`game_design.md`, `docs/agents/*.md`, `docs/design/*.md`)뿐.
- 검증 결과 창작, 완료 격상, 지시 없는 커밋/푸시(커밋은 제작자 지시 시 `msw-checkpoint` 스킬 사용).

## 1. 공통 제작 규칙 (모든 아트 티켓에 적용)

1. **크기**: 1 world unit = 100px. 마을 건물 **400~700px 폭**(4~7유닛), 소품 50~250px, 인벤토리 아이콘 **32~64px**(원본 도트 그대로, 확대 금지).
2. **구도**: 이 게임은 톱다운(RectTile)이지만 JRPG 관례상 **정면(front-view) 건물 배치가 성립**한다. MSW 원본(측면 사이드뷰용)을 쓸 때 image-to-pixel 리드로우에서 다음을 손본다:
   - **접지선 정리** — 밑변을 수평 기준선으로, 드롭섀도는 납작한 타원으로 재작업 (사이드뷰 그림자·비탈 제거).
   - **높이 15~25% 압축 + 지붕 상면 비중 확대** — 톱다운 시점감.
   - **픽셀 밀도 통일** — 후보 중 벡터 카툰풍(예: 여관 8bf7997e)은 도트 리드로우, 도트풍(예: 70993f0d)은 팔레트만 보정.
   - **팔레트** — 기존 맵 잔디/흙 톤(T24·T51 기준)과 채도 맞춤.
3. **작업 절차 (RUID 소스 → 게임 반영)**:
   1) `getResource(ruid)` → `payload.thumbnail` URL에서 원본 PNG 확보 (`.claude/skills/msw-search/scripts/msw_resource_api.cjs`)
   2) `image-to-pixel` 스킬로 리드로우(구도 변경) — 원본을 그대로 쓸 수 있으면 이 단계 생략하고 RUID 직접 바인딩
   3) 신규 이미지는 `msw-painter` 스킬의 업로드 절차(msw-mcp `asset_create_*`)로 **신규 RUID 취득**
   4) 건물/소품 = `.model` 신규 작성(ModelBuilder, `MapObject` 템플릿) + `town.map` 배치(MapBuilder) / 아이콘 = `item_dataset.csv` IconRUID 셀 교체
4. **모델 규칙**: 같은 소품 2개 이상 배치 시 반드시 `.model` + `modelId` 인스턴스 (AGENTS.md §3-11). 저장 위치 `RootDesk/MyDesk/MapObjects/Models/`.
5. **비충돌 데코**는 Body 불요. 통행 차단할 건물은 기존 `Building_Shop` 구성을 미러.

# Phase 15 기획서 — 살아있는 월드 확장 (Living World Expansion)

> **작성**: 지휘자, 2026-07-11. 구현 티켓: `docs/agents/subagent-handoff.md` §3 **T16~T23** (배치 실행 가이드 포함).
> **위상**: Phase 14(농사·연구·침대·희귀드롭)가 "수직 진행"을 채우는 동안, Phase 15는 **가로 방향 — 매일 돌아오게 만드는 생활 루프와 살아있는 월드**를 채운다.

---

## 1. 문제 인식 — 지금 게임에 부족한 것

현재 루프: 채집 → 제작 → 제련 → 건설/꾸미기 → 사냥 → 보스. Phase 14 완료를 가정해도 다음 5개가 비어 있다.

| # | 공백 | 증상 |
|---|---|---|
| 1 | **산출물의 소비처 부족** | 작물·고기·(예정)생선이 인벤토리에 쌓이기만 함 — "먹는" 시스템이 없다 |
| 2 | **일일 접속 훅 부재** | 오늘 접속할 이유가 없다 — 리텐션 장치 0 |
| 3 | **성장 외 목표 부재** | 수집/기록/과시 메타 없음 — 테크를 다 올리면 할 일이 끝남 |
| 4 | **월드가 정적** | 낮/밤은 있지만 날씨·변화 없음 — 어제와 오늘의 월드가 동일 |
| 5 | **제3의 액티비티 부재** | 조작이 채집(Ctrl)과 전투뿐 — 쉬어가는 놀이(낚시류)가 없다 |

## 2. 디자인 원칙

1. **재사용 우선**: 모든 신규 시스템은 기존 파이프라인(가구 설치·드롭·서버 타이머·F 상호작용·영지 델타 영속화)을 재사용한다. 신규 발명은 버프 인프라(T16) 하나뿐.
2. **CSV = 콘텐츠**: 모든 밸런스·콘텐츠 값은 데이터셋 행 — 행 추가만으로 음식/어종/가축/의뢰/날씨/업적이 늘어난다 (AGENTS.md §2 하드코딩 금지의 적극적 활용).
3. **루프가 서로 물리게**: 단품 기능 금지. §4 통합 루프에서 각 시스템이 기존 시스템의 입력/출력이 되어야 한다.
4. **영지는 평화 유지**: §2.2 ① 원칙(전투 몹 금지) 불변 — 가축·펫·낚시는 전부 수동적.

## 3. 시스템 기획 (8종)

### 3.1 공통 버프 시스템 (T16) — 선행 인프라 ⚙️

- **왜**: 요리(음식 버프)·날씨·펫이 전부 "일시적 스탯 변화"를 요구한다. 각자 구현하면 3벌 중복 — 공통 모듈 하나로.
- **무엇**: `BuffDataSet.csv`(BuffId/DisplayName/StatKey/ModifierType[add|mult]/Value/Duration/IconRUID/StackRule[refresh|ignore]) + 플레이어 버프 컴포넌트(적용/만료/스탯 재계산) + HUD 버프 아이콘 바(남은 시간).
- **스탯 키(초기)**: `MoveSpeed`(이동속도), `GatherSpeed`(채집속도), `AttackPower`(공격력), `StaminaRegen`(스태미나 회복). 소비처는 각각 기존 시스템의 값 산출 지점에 배율/가산 1줄 훅.
- **소비 아이템 경로**: `item_dataset`에 `UseBuffId` 컬럼 + `Category=consumable` 신설 — 퀵슬롯에서 사용(기존 tool=장착/furniture=설치와 나란한 세 번째 데이터 주도 분기) → 수량 1 차감 → 버프 적용.
- **MVP 단순화(⚖️ 확정)**: 버프는 **세션 한정**(로그아웃 시 소멸, 영속화 안 함). 중첩은 같은 BuffId면 시간 갱신(refresh)이 기본.

### 3.2 요리 (T17) — 농사·사냥·낚시의 소비처 🍲

- **왜**: 공백 #1의 직접 해결. 작물(T6)·고기(사냥)·생선(T18)의 사용처이자 버프의 공급처.
- **무엇**: 설치 가구 **조리 냄비(Cooking Pot)**. 화로(`Furnace.mlua`) 파이프라인을 그대로 쓰되, **레시피 테이블 이름을 컴포넌트 프로퍼티로** 받게 소폭 일반화 — 화로=제련 테이블, 냄비=`CookingRecipeDataSet`. 연료도 기존 `FurnaceFuelDataSet` 공유.
- **음식 = 버프 아이템**: 결과물은 `Category=consumable` + `UseBuffId`. 초기 5종 예시 — 구운 고기(공격력↑), 당근 수프(채집속도↑), 생선 스튜(스태미나 회복↑), 야채 볶음(이동속도↑), 잔치 요리(상위 버프·재료 다수).
- **루프**: 밭/사냥/낚시 → 재료 → 요리 → 버프 → 더 효율적인 사냥/채집 → 더 좋은 재료.

### 3.3 낚시 (T18) — 제3의 액티비티 🎣

- **왜**: 공백 #5. 채집·전투와 완전히 다른 리듬(대기→반응)의 원버튼 놀이. 코어키퍼/스타듀의 검증된 휴식 루프.
- **무엇**: 낚싯대(도구, `ToolType=rod`) + **낚시터 픽스처 엔티티**(영지 연못·마을 분수·사냥터 물가 — 맵에 배치, 타일 지형은 건드리지 않음) + `FishDataSet.csv`(FishItem/SpotType/Weight/BiteTimeMin·Max/Rarity).
- **미니게임(원버튼 타이밍)**: 낚시터에서 F → 캐스팅 → 랜덤 대기(2~6초) → 머리 위 `!` 표시 → **0.8초 안에 다시 입력**하면 어획, 놓치면 재대기. 이동 시 취소. 화려한 게이지 미니게임은 비범위(후속 고도화).
- **어종 차등**: SpotType(estate/town/field)별 어종 풀 — 사냥터 물가에서만 나오는 희귀어(요리 상위 재료·도감 항목).

### 3.4 목장/가축 (T19) — 영지의 두 번째 생산 축 🐔

- **왜**: 농사(작물)와 대칭인 **동물 생산** 축. 영지에 "돌봐야 할 생명"이 생기면 애착과 재방문 동기가 생긴다.
- **무엇**: **우리(Animal Pen)** 가구 + 가축 구매(마을 상점) + `AnimalDataSet.csv`(AnimalId/FeedItem/FeedInterval/ProduceItem/ProduceInterval/SpriteRUID). 초기 2종 — 닭(먹이=씨앗류, 산출=달걀), 양(먹이=풀, 산출=양털).
- **동작**: 가축은 우리 반경 내 배회(수동적 — 몬스터 wander 로직 재사용, 전투 없음). 먹이를 들고 F → 급여 → 시간 경과 후 산출물이 근처에 드롭(기존 드롭 파이프라인).
- **루프**: 씨앗/풀(농사) → 먹이 → 달걀/양털 → 요리 재료·제작 재료(양털→침구/장식 후속).

### 3.5 마을 의뢰 게시판 (T20) — 일일 훅 📋

- **왜**: 공백 #2의 최소비용·최대효과 해결. "오늘의 의뢰"는 매일 접속할 이유 + 잉여 자원의 소비처 + 마을에 갈 이유.
- **무엇**: 마을에 **게시판(Bulletin Board)** 픽스처 + `RequestPoolDataSet.csv`(RequestId/RequiredItem/RequiredCount/RewardItem/RewardCount/Weight).
- **일일 결정론**: 서버 날짜(일 번호)를 시드로 풀에서 **3건 추첨 — 전 서버 공통**(모든 유저가 같은 의뢰를 보므로 커뮤니티 대화거리가 된다). 유저별 의뢰당 하루 1회 납품, 완료 기록 영속화, 자정(서버 일 변경) 리셋.
- **보상 설계**: 코인 중심 + 낮은 확률로 희귀 재료/도안(T9 계층 연결). 요구 품목에 농사/낚시/가축 산출물을 섞어 **Phase 15 콘텐츠의 수요처**로 쓴다.

### 3.6 날씨 (T21) — 살아있는 월드 🌧️

- **왜**: 공백 #4. 어제와 다른 오늘. 낮/밤(구현 완료)과 직교하는 두 번째 월드 상태 축.
- **무엇**: `WeatherDataSet.csv`(WeatherId/Weight/DurationMin·Max/OverlayColor/EffectRUID/CropBoostPerMin/FishBiteMult) + 서버 날씨 매니저(주기 롤·@Sync 브로드캐스트) + 클라 오버레이(밤 오버레이 연출 패턴 재사용). 초기 3종 — 맑음/비/안개.
- **게임플레이 훅**: ① 비 = 영지 작물 성장 가속(1분마다 plantedAt을 N초 당기는 서버 틱 — 타임스탬프 모델과 정합) ② 비 = 낚시 입질 배율↑ ③ 안개 = 시야 연출(순수 무드). 몬스터 연동은 비범위.
- **원칙**: 날씨는 **보너스만** 준다(페널티 없음) — 비 오는 날 접속하면 이득, 안 와도 손해 없음.

### 3.7 도감 & 업적 (T22) — 수집 메타 📖

- **왜**: 공백 #3. `game_design.md` §3.11에 제안만 있던 도감의 실행. 수집 완성도가 장기 목표가 된다.
- **무엇**: ① **통계 카운터 인프라** — 아이템 최초 획득/누적 획득, 몬스터 처치, 제작 횟수를 유저별 영속 기록(획득·제작·처치 경로에 1줄 훅) ② **도감 UI** — T14 도감형 골격 재사용(탭: 아이템/몬스터/업적, 미발견=실루엣) ③ `AchievementDataSet.csv`(AchievementId/ConditionType[collect|kill|craft|fish...]/ConditionTarget/ConditionValue/RewardItem/RewardCount) — 달성 시 보상 수령 버튼.
- **자동 확장**: 도감 항목은 `item_dataset`/몬스터 데이터에서 파생 — 신규 아이템·어종·가축이 추가되면 도감이 자동으로 늘어난다.
- **선택 연계(보류 가능)**: MSW `world_badge`(플랫폼 배지)와 대표 업적 연동은 후속 — 데이터 컬럼(`BadgeId`)만 예약.

### 3.8 펫 동반자 (T23) — 편의 + 애착 🐕

- **왜**: 파밍 편의(줍기 자동화)를 "귀여움"으로 포장한 보상형 콘텐츠. 보스 드롭/업적 보상의 매력적인 사용처.
- **무엇**: `PetDataSet.csv`(PetId/SpriteRUID/MoveSpeed/PickupRange) + 펫 소환 아이템(사용=활성 펫 교체) + 팔로워 엔티티(플레이어 추종, 주변 드롭 자동 픽업 — 기존 자석 픽업 파이프라인을 펫 위치 기준으로 재사용). 활성 펫 영속화.
- **입수처**: 첫 펫(강아지)=온보딩 퀘스트 또는 상점, 희귀 펫=보스 드롭/업적 보상(T9·T22 연결). 전투 없음 — 순수 동반자.

## 4. 통합 루프 — 시스템이 서로 물리는 그림

```
        ┌────────── 영지 ──────────┐          ┌── 마을 ──┐          ┌── 사냥터 ──┐
  괭이→밭→작물 ─┐                                │
  우리→가축→달걀·양털 ─┤→ 요리(냄비) → 음식 버프 ─┼→ 사냥 효율↑ → 전리품·희귀어·도안
  연못→낚시→생선 ─┘         ↑                    │                    │
        ↑                비(날씨): 성장·입질 가속  │                    ↓
        └── 의뢰 게시판(일일): 작물·생선·달걀 납품 → 코인·희귀 보상 ← 연구소 해금
                            ↓
              도감·업적: 전 활동을 기록 → 보상·펫 → 파밍 편의 → 반복
```

- 매일: 날씨 확인 → 물 준 셈 치는 비 보너스 → 의뢰 3건 → 수확·급여·낚시 15분 루프.
- 매주: 도감 빈칸 채우기(희귀어·희귀 변종·보스 도안) → 업적 → 펫/칭호.

## 5. 실행 계획 — 배치·의존성 (구현은 전부 하위 에이전트 위임)

```
배치 A (Phase 14 잔여 — UI/진행 골격):  T14 → T7 → T8 → T9   (T15 병행 가능)
배치 B (생활 루프 1차):               T16 → T17 → T20 → T18
배치 C (살아있는 월드 2차):            T19 → T21 → T22 → T23
```

- **의존성 하드 제약**: T7←T14(UI 골격), T9←T7(해금 계층), T17←T16(버프)+T6(작물), T21←T6(작물 훅)·T18(입질 훅), T22←T14(UI 골격), T23←T16(자석/사용 경로 정리 후 권장).
- **배치 내 순차가 기본** — 신규 태스크들이 `PlayerController`/`PlayerInventory`/`PersistenceManager`/`item_dataset.csv`를 공유한다. 병렬은 다른 머신+브랜치에서만, 태스크별 파일 소유권은 handoff §3 각 항목의 "충돌 주의" 참조.
- 상세 스펙·완료 기준은 전부 `docs/agents/subagent-handoff.md` §3 T16~T23.

## 6. 명시적 비범위 (이번 Phase에서 하지 않는 것)

- **영지 방문·랭킹/좋아요** (꾸미기 과시 메타) — Phase 16 유력 후보. 방문 동선 재설계(§2.2 포탈)와 리더보드 패키지 검토 필요.
- **계절 테마** (§2.2 ① 예정) — 타일셋 리스킨 물량 작업이라 아트 확정 후.
- **낚시 게이지 미니게임 고도화** — 원버튼 타이밍으로 시작, 반응 보고 고도화.
- **가축 번식/성장 단계**, **펫 스킬/전투**, **날씨-몬스터 연동**, **허기(음식 강제) 시스템** — 허기는 페널티형이라 현재 톤(아늑한 생활)과 충돌, 도입하지 않음(⚖️ 2026-07-11 기각).

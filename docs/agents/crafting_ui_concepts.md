# 아이템 연구 및 제작 UI/UX 컨셉 제안

아이템 종류가 많아지고 '연구를 통한 해금'이라는 시스템이 추가될 때, 단순 리스트 방식은 스크롤 압박과 직관성 저하를 유발합니다. 이를 해결하기 위한 몇 가지 UI 방식을 제안합니다.

## 1. 테크 트리 (Tech Tree / Node-based) 방식
가장 직관적으로 "선행 연구"와 "다음 단계"를 보여주는 방식입니다. (예: 문명, 팩토리오, 패스 오브 엑자일 등)

- **장점**: 
  - 어떤 아이템을 해금하기 위해 무엇이 필요한지 한눈에 파악 가능.
  - 유저에게 명확한 목표성(다음 노드 해금)을 제공.
- **단점**: 
  - 아이템이 너무 많아지면 화면이 복잡해짐.
  - MSW 기본 UI 컴포넌트로 구현 시 위치 계산이나 선 연결(Line) 처리가 다소 까다로울 수 있음.
- **MSW 구현 방안**: 
  - 큰 `ScrollLayoutGroup` (Free 이동 가능하게 세팅) 위에 노드(버튼)들을 절대 좌표로 배치. 
  - 선 연결은 얇고 긴 픽셀 이미지의 `Transform.Rotation`과 크기를 조절하여 노드 사이를 잇는 방식으로 구현.

## 2. 카테고리 탭 + 그리드 뷰 (Category Tabs + Grid View)
마인크래프트나 테라리아, 일반적인 RPG 인벤토리처럼 탭으로 대분류를 나누고 아이템을 바둑판 배열로 보여주는 방식입니다.

- **장점**: 
  - 아이템이 아무리 많아도 탭으로 분산시켜 깔끔하게 수납 가능.
  - MSW의 `GridView` 컴포넌트를 사용해 매우 쉽고 깔끔하게 구현 가능.
- **단점**: 
  - "이 아이템을 해금하려면 어떤 연구를 먼저 해야 하는지" 선후 관계를 한눈에 보기는 어려움 (상세 정보 창을 띄워야 알 수 있음).
- **해결책**: 
  - 잠긴 아이템은 자물쇠 아이콘이나 흑백 처리(Color 틴트 조정)를 하고, 클릭 시 우측 상세 패널에 "선행 조건: 철 괴 연구 완료" 등을 텍스트로 명시.

## 3. 도감형 (Blueprint Book) 방식
화면을 좌우로 분할하여, 좌측에는 카테고리별 그리드(목록)를 두고 우측에는 선택한 아이템의 상세 정보를 크게 보여주는 방식입니다.

- **장점**: 
  - 화면 공간을 효율적으로 사용하며, 상세 정보를 충분히 보여줄 수 있음.
  - 리스트의 직관성과 디테일한 정보를 동시에 잡는 가장 표준적이고 모던한 UI.
- **MSW 구현 방안**: 
  - 좌측: `ScrollLayoutGroup` 내부 상단에 탭을 두고 아래에 `GridView` 배치.
  - 우측: 선택된 아이템의 큰 썸네일(Image), 이름, 설명 텍스트, 요구 재료 목록, 선행 연구 조건, 최종 **[연구하기] / [제작하기]** 버튼 배치.

## 4. 진행도 기반 티어(Tier) 분리 방식
연구 단계를 시대나 티어(Tier 1, Tier 2, Tier 3...)로 나누어, 현재 해금된 티어의 아이템만 화면에 보여주는 방식입니다.

- **장점**: 
  - 초보 유저에게 너무 많은 아이템을 한 번에 보여주어 발생하는 정보 과부하(Information Overload) 방지.
  - "다음 티어 해금"이라는 굵직한 게임 플레이 목표 제공.
- **단점**: 
  - 상위 티어에 어떤 아이템이 있는지 구경하고 기대감을 갖게 하려면, 실루엣 처리 등 별도의 '미리보기' 연출이 필요함.

## 💡 추천 조합 제안 (하이브리드)

MSW 환경에서 개발 효율성과 유저 편의성을 모두 잡을 수 있는 **[도감형 + 티어 분리 + 그리드]** 결합 모델을 추천합니다.

1. **상단 탭 (티어/시대 분류)**: 
   - `[석기 시대]` `[청동기 시대]` `[철기 시대]` 
   - (해금되지 않은 탭은 자물쇠 표시)
2. **좌측 패널 (아이템 목록)**: 
   - 탭 내에서 카테고리 필터(`[전체] [도구] [자재]`) 제공.
   - 아래는 `GridView`로 아이템 아이콘 나열. (미해금은 자물쇠/실루엣)
3. **우측 패널 (상세 정보)**: 
   - 클릭한 아이템의 큼직한 아이콘과 설명.
   - **요구 재료**: 아이콘 + `(보유량/필요량)` 텍스트.
   - **선행 조건**: "돌도끼 제작 5회 완료" 또는 "이전 티어 연구 필요" 등 명시.
   - 하단에 **[연구하기(포인트 소모)]** 또는 **[제작하기]** 버튼 활성화.

이 방식은 테크 트리처럼 복잡한 선 연결 연산이 필요 없어 버그 발생 확률이 낮고, 훗날 아이템이 수백 개로 늘어나도 UI 레이아웃이 깨지지 않는다는 강력한 장점이 있습니다.
